# 前端开发原型合规流程

本规范源于一期/二期开发中反复出现的问题：前端组件开发完成后，与原型HTML存在大量遗漏。
根本原因是开发时只按任务描述写功能逻辑，没有逐元素对照原型。以下流程必须严格执行。

## 开发前：原型解析（必做）

1. 完整读取原型HTML文件（skipPruning=true），不能只读部分
2. 针对当前要开发的组件/页面，从原型中提取完整的元素清单：
   - 页面结构（几个区块、层级关系、吸顶行为）
   - 每个卡片/区块内的所有子元素（标题、标签、数值、按钮、图标）
   - 交互行为（展开/收起、点击跳转、弹窗）
   - 颜色值、字号、间距等视觉参数
3. 将提取的元素清单作为开发的checklist，不是任务描述

## 开发中：逐元素实现

对照checklist逐项实现，每个元素必须能在代码中找到对应：
- 原型里有的按钮 → 代码里必须有对应的button + onClick
- 原型里有的数据展示 → 代码里必须有对应的数据获取 + 渲染
- 原型里有的颜色/样式 → 代码里必须用相同的色值
- 原型里直接可见的信息 → 不能藏在展开/折叠后面（除非原型本身就是折叠的）

## 开发后：逐项核对（必做）

完成编码后，必须执行以下核对步骤，不能跳过：

### 1. 结构核对
- 原型的页面数量 = 实现的路由数量
- 原型的底部导航tab数 = BottomNav的tab数
- 原型每个页面的区块数 = 实现的组件数

### 2. 元素核对（逐卡片）
对原型中每个卡片/区块，检查：
- [ ] 所有文字标签都已渲染
- [ ] 所有数值字段都已绑定数据
- [ ] 所有按钮都已实现（包括操作按钮如"加入关注"、"快速买入"）
- [ ] 所有标签/徽章都已显示（估值标签、信号标签、置信度标签）
- [ ] 所有可展开区域的默认展开/收起状态与原型一致

### 3. 视觉核对
- [ ] 颜色值与原型一致（特别是深色背景上的标签颜色）
- [ ] 信息层级与原型一致（哪些直接可见、哪些需要展开）
- [ ] 按钮样式与原型一致（填充色、边框色、圆角）

### 4. 测试核对
- [ ] 新增/修改的组件有对应的测试文件
- [ ] 测试中的断言值（颜色、文本）与实际实现一致
- [ ] 运行测试套件确认全部通过

## 常见遗漏模式（必须警惕）

这些是历史上实际发生过的遗漏，开发时必须主动检查：

1. **按钮遗漏**：原型卡片底部有操作按钮（加入关注、快速买入、深度报告、历史回测），实现时漏掉或藏在展开面板里
2. **信息层级错误**：原型中直接可见的技术信号标签、持仓天数、预估收益，被错误地放到了需要点击展开才能看到的面板里
3. **页面合并**：原型是独立的两个页面（如看板和持仓），实现时错误地合并成一个页面
4. **导航数量不匹配**：原型有5个底部tab，实现时只做了4个
5. **颜色不匹配**：组件颜色更新后，测试文件中的颜色断言没有同步更新
6. **深色背景适配**：标签从浅色背景移到深色背景后，颜色没有相应调整

## 完成标准

只有满足以下全部条件，才能标记前端任务为完成：
1. 原型元素清单100%覆盖（无遗漏元素）
2. 所有测试通过
3. 无TypeScript诊断错误
