# 项目经验教训（一期总结，二期必须遵守）

本文件总结了一期开发中用户反馈的所有问题，开发时必须主动避免这些问题。

## 数据接口规则

- 东方财富 push2his 接口不可访问，历史K线必须用腾讯 `web.ifzq.gtimg.cn`
- 新浪/腾讯行情接口返回 GBK 编码，必须用 `responseType: 'arraybuffer'` + `TextDecoder('gbk')` 解码
- AI provider 使用 class 语法：`new DeepSeekProvider('key')`
- DeepSeek 模型名：`deepseek-chat` = V3（默认），`deepseek-reasoner` = R1（慢）
- 不要读取 `server/.env`，只能通过 append 方式修改

## AI输出措辞合规

- 所有AI输出必须使用"参考方案"措辞，禁止使用"建议"/"推荐"（功能名称如"埋伏推荐"除外）
- 复盘评价不能包含批评或指责性语言

## 前端UI规则

- 前端页面不显示"一期"/"二期"版本标签
- 用户输入应为单一搜索框（名称或代码），不要分成两个输入框
- 吸顶区域要紧凑，不要浪费空间
- 持仓页标题+tab应作为一个整体吸顶
- 事件日历需要清晰可操作的标签，不要用晦涩术语如"出货窗口"，用"利好兑现·可减仓"
- 关注tab的卡片需要和持仓卡片一样的完整分析功能
- 周期监控描述必须清楚说明：周期节奏、当前位置、已经持续多久、距离下一阶段还有多久
- 消息中心筛选标签要精简合并，不要太多分类
- 通知设置应该是独立子页面，有返回按钮和开关控件
- 移动端H5优先，所有交互要适配手机屏幕

## UI设计高标准（世界顶级UI设计师水准）

- 字体清晰度：正文最小14px，辅助文字不低于12px，标签文字至少12px且padding充足
- 对比度：浅色文字必须保证足够对比度，深色背景上文字用纯白或高亮色，避免灰色文字在浅灰背景上
- 间距节奏：卡片内padding 16-20px，元素间距8-12px，保持呼吸感但不浪费空间
- 色彩层次：延续一期紫靛渐变设计系统，状态色要鲜明（涨红跌绿、低估绿/合理蓝/高估红）
- 圆角统一：卡片12-16px，标签6-8px，按钮8-12px
- 阴影层次：卡片用柔和阴影（0 2px 12px rgba(0,0,0,0.06)），悬浮元素加深
- 动效：过渡动画0.2-0.3s ease，tab切换、弹窗出现要有流畅过渡
- 图标与文字对齐：emoji/图标与文字垂直居中，不要错位
- 触摸友好：所有可点击元素最小44x44px触摸区域
- 信息密度：金融数据展示要紧凑但不拥挤，关键数字要突出（大字号+粗体+强调色）
- 空状态：所有列表/数据区域要有优雅的空状态设计，不要留白
- 加载状态：数据加载时用骨架屏或优雅的loading动画，不要空白闪烁

## 后端架构规则

- 二期是在一期代码基础上迭代，扩展现有文件、新增模块，不要重写一期代码
- 定时任务必须先判断是否为A股交易日（周末+法定节假日跳过）
- 盘中定时分析仅在交易日 9:30-11:30、13:00-15:00 运行（午休11:30-13:00不触发）
- 收盘后定时任务在 15:30-17:30 错峰串行执行
- 服务器是阿里云轻量 2核2G，注意内存和CPU占用
- 本地开发：后端端口3000，前端端口5173 + `--host`

## 测试规则

- 服务端测试：`npx jest --no-coverage`（在 server/ 目录下）
- 客户端测试：`npx jest --no-coverage`（在 client/ 目录下）
- fast-check 已安装在 server 和 client（`"fast-check": "^3.23.0"`）
- 属性测试必须通过注释引用设计文档中的属性编号

## 降本优化架构（二期核心）

- 大盘环境判断、持仓集中度、操作复盘评价 = 纯规则，零AI调用
- 止损评估 = 规则为主，AI按需（用户点击时）
- 自我修正 = 仅记录偏差事实，不调用AI
- 定时分析采用"规则先行 + AI批量润色"模式：规则引擎算结构化数据，一次批量AI调用生成所有reasoning文本+埋伏推荐文本+每日关注文本
- 变化触发制：股价变化<2% 且 RSI变化<5 → 跳过AI，复用上次结果
- 股票去重：多用户持同一股票只分析一次
- 深度报告24h缓存跨用户共享
- Prompt瘦身：只传关键指标+新闻标题
